## Aprendizajes

### 1. ¿Qué fue lo más fácil y lo más retador?

Lo más fácil fue crear la estructura básica utilizando HTML y organizar el contenido con etiquetas semánticas.
Lo más retador fue implementar la reproducción de audio, el sistema de calificación con estrellas y conectar correctamente el repositorio con GitHub Pages para publicar el sitio.


### 2. ¿Qué etiquetas semánticas usaste y por qué?

Utilicé las siguientes etiquetas semánticas:

* `header` para el encabezado y navegación
* `main` para el contenido principal
* `section` para separar cada parte de la página
* `article` para los artistas
* `footer` para la información final

Estas etiquetas ayudan a organizar mejor el contenido y mejorar la estructura del sitio.


### 3. ¿Cómo organizaste tus commits?

Realicé commits cada vez que agregaba una nueva funcionalidad o cambio importante, por ejemplo:

* Crear la estructura inicial del proyecto
* Agregar la playlist
* Agregar imágenes y estilos
* Agregar reproducción de audio
* Agregar sistema de calificación con estrellas
* Actualizar el README y evidencias

Esto me permitió mantener un historial claro del desarrollo del proyecto.


### 4. ¿Qué mejorarías en la siguiente iteración?

Me gustaría mejorar el diseño visual agregando más animaciones y efectos con CSS.
También me gustaría agregar más funcionalidades, como guardar las calificaciones del usuario y hacer el diseño completamente responsive para dispositivos móviles.
